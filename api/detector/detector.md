Задача модели определить CMS сайта по одной главной странице (homepage) с максимальной достоверностью на основе анализа сетевых и HTML-артефактов, а не визуального контента
Входные данные полный HTTP-трафик загрузки главной страницы включая HTML response headers cookies список всех загруженных ресурсов (CSS JS images fonts) и их URL а также inline данные в HTML
Модель должна анализировать только фактически полученные данные без выполнения дополнительных запросов (никакого обхода /admin /wp-login и т.д.)
Основной принцип детектирования использовать множественные независимые маркеры так как ни один метод не дает 100% результата в одиночку 
Модель должна собирать признаки из следующих источников HTTP headers cookies HTML source code asset URLs inline JS данные комментарии и служебные теги так как именно эти зоны содержат сигнатуры CMS 
Категории сигналов разделить по силе
Сильные сигналы прямое указание CMS в meta generator X-Generator X-Powered-By наличие уникальных CMS cookie (например wordpress_test_cookie) совпадение нескольких уникальных путей ассетов
Средние сигналы характерные пути к ресурсам (например /wp-content /sites/default /cdn.shopify.com) специфические link rel API endpoints inline JS конфигурации содержащие названия платформ
Слабые сигналы структура HTML классов DOM паттерны UI компоненты общие JS библиотеки
Модель обязана проверять HTTP headers на наличие X-Powered-By X-Generator Server и других нестандартных заголовков учитывая что они могут быть скрыты или подменены 
Модель обязана анализировать cookies и искать сигнатурные имена так как это один из наиболее надежных способов fingerprinting 
Модель обязана анализировать HTML на наличие meta generator комментариев служебных тегов inline переменных и специфичных путей
Модель обязана анализировать все asset URLs включая CDN домены структуру путей параметры версионирования
Модель должна учитывать что современные сайты могут использовать headless архитектуру при которой CMS не оставляет следов в HTML и тогда детект невозможен
Модель должна учитывать что признаки могут быть намеренно скрыты обфусцированы или подделаны поэтому отсутствие признака не является доказательством отсутствия CMS 
Модель должна исключать ложные срабатывания от сторонних виджетов и embed компонентов учитывая только признаки относящиеся к основному документу
Модель должна агрегировать признаки и принимать решение на основе их комбинации а не одного совпадения
Правило уверенности
если найден прямой маркер CMS (generator или header с названием CMS) и он не противоречит другим данным вывод CMS определена с высокой достоверностью
если найдено минимум два независимых сильных сигнала вывод CMS определена с высокой вероятностью
если найден один сильный или несколько средних сигналов вывод CMS вероятна
если найдены только слабые сигналы вывод CMS не может быть достоверно определена
если отсутствуют сигналы вывод CMS не определяется по главной странице
Модель должна возвращать не только название CMS но и список обнаруженных сигналов с классификацией по силе
Модель не должна делать выводы на основе предположений или шаблонных догадок без фактических артефактов
Модель должна явно указывать уровень уверенности результата
Модель должна быть устойчива к ложным паттернам и учитывать что fingerprinting основан на утечках информации и совпадении сигнатур 
Если коротко твоя “идеальная” модель не угадывает CMS, она собирает улики как нормальный криминалист и потом выносит приговор с вероятностью, а не с пафосом.
